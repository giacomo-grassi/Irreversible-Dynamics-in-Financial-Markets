# data/ — Historical Prices & Rolling Estimation Outputs

This directory contains the raw financial time series and the processed rolling-window outputs used in the empirical analysis of scaling dynamics and financial stress.

The structure follows a clear separation between:

- **Raw market data**
- **Derived statistical estimates**

---

---

## 📊 raw/

### PRICE_DEF.txt

Historical daily closing prices of the **Dow Jones Industrial Average (DJIA)**.

Format:

| Date | Close |
|------|-------|
| YYYY-MM-DD | Price |

The dataset begins in **1896** and provides the long historical span required to study scaling behavior and structural regime changes in financial markets :contentReference[oaicite:1]{index=1}.

This file represents the *primary market input* for the entire analysis.

---

## ⚙️ processed/

Contains outputs generated programmatically through the rolling estimation pipeline implemented in `/src`.

### rigid_windows_results.csv

Rolling-window estimates of scaling parameters and related statistics over the full sample.

Includes:

- Estimated β (ACF decay exponent)
- Estimated Dₑ (effective anomalous diffusion exponent)
- Stress proxy alignment
- Additional diagnostics

---

### rigid_windows_results_125.csv

Rolling estimation results using:

- Window size: **125 trading days**
- Shift: fixed step (rigid rolling)

This specification is used for baseline inference and statistical testing in the main empirical analysis.

---

## 🔄 Data Processing Pipeline

The transformation flow is:

Raw DJIA Prices  
→ Log Returns  
→ Rolling Window Construction  
→ Parameter Estimation (β, Dₑ)  
→ Stress Alignment  
→ Statistical Testing

All derived files were generated through reproducible code.  
No manual adjustments were applied.

---

## 📌 Reproducibility

- Rolling estimation is deterministic given fixed parameters.
- All results can be regenerated by executing the pipeline in `/src`.
- The processed files serve as saved outputs for verification and transparency.

---

## 🎯 Conceptual Role of This Folder

The `data/` directory represents the empirical backbone of the project:

- `raw/` encodes the historical market trajectory.
- `processed/` encodes the dynamic structural state variables extracted from that trajectory.

Together, they allow investigation of whether scaling parameters behave as macro-financial stress indicators.
